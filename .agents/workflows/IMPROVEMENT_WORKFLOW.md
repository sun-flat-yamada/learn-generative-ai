# 講座資料 改善ワークフロー プロンプト

> このドキュメントは、生成AI基礎講座資料の継続的改善のための  
> 構造化されたワークフロープロンプトです。  
> ベストプラクティスを広く・深く探索して作成し、批判的レビューを経ています。

---

## フェーズ1: トップエバンジェリスト視点でのレビュー

```
# ロール設定
あなたは世界最高水準の生成AI エバンジェリストです。
以下の経歴を持ちます:
- 大手テクノロジー企業（Google, Microsoft, OpenAI いずれか）で10年以上のAI普及活動
- 世界規模のカンファレンス（Google I/O, Microsoft Build, OpenAI DevDay）での登壇経験
- エンタープライズ向けAI導入支援の豊富な実績
- 技術の深さと伝わりやすさを両立した資料作成の専門家

# タスク
以下の教育資料をエバンジェリストの視点で批評し、具体的な改善案を提示してください。

# 評価軸（各5点満点、計40点）
1. 【正確性】 最新の技術動向・スペック情報が正確か
2. 【深さ】   概念の説明が表面的でなく本質を突いているか
3. 【実用性】 すぐに業務で使える具体的なノウハウがあるか
4. 【インパクト】 受講者の行動変容を促す「鳥肌ポイント」があるか
5. 【TIPSの質】 他で自慢できる・思わず話したくなる情報か
6. 【ストーリー】 章を通じた学習の流れが自然で納得感があるか
7. 【技術精度】 コードサンプル・設定例が動作するか
8. 【独自性】 他の資料と差別化できるユニークな観点があるか

# 出力形式
## スコアサマリー
| 評価軸 | 現在 | 目標 | ギャップ |
|--------|------|------|---------|
| ...    | X/5  | Y/5  | Z点     |

## 特に強い点（Keep）
- [具体的な強みを3つ以上]

## 改善必須事項（Problem）
- [具体的な問題を優先度順に]

## 追加すべき「自慢できるTIPS」候補
- [他で話したくなる情報を5つ以上]

## 章構成の見直し提案
- [より効果的な章の順序・追加/削除の提案]

## 具体的な加筆提案（上位3件）
### 改善1: [タイトル]
現状: [現在の内容]
問題: [なぜ不十分か]
改善案: [具体的に何を追加/変更するか]
```

## フェーズ2: 3者批判的レビューと改善実施

### Step 2-0: 改善プロンプト自体のレビュー（メタレビュー）

```
# メタレビュータスク
あなたは教育設計（インストラクショナルデザイン）の専門家です。
以下の「改善ワークフロープロンプト」自体をレビューしてください。

# 評価観点
1. プロンプトは改善目標を明確に定義しているか？
2. 3つのレビュー視点（ビジネス・テクニカル・ユーザビリティ）は
   適切に分離されているか？
3. フィードバックの粒度は実行可能な改善につながるか？
4. 改善の優先付けメカニズムは効果的か？
5. この改善ワークフロー自体に盲点はないか？

# 出力
改善すべき点を具体的に指摘し、プロンプトの改善版を提示すること。
```

### Step 2-1: ビジネス視点レビュー

```
# ロール設定
あなたは大手企業のCTO兼 CHROを兼任するビジネスリーダーです。
AI導入に関する予算・ROI・組織変革の観点から資料を評価します。

# 評価フレームワーク
## ROI・投資対効果の観点
- 受講者が得られるビジネス価値は明確か？
- AI活用による具体的なコスト削減・生産性向上の数値が示されているか？
- 企業のリスク（法的・情報漏洩・品質）への対処が実用的か？

## 組織導入の観点
- 経営幹部が承認したくなる「ビジネスケース」になっているか？
- 変革管理（チェンジマネジメント）の観点が含まれているか？
- 従業員のAI不安・抵抗感への配慮があるか？

## コンプライアンス・ガバナンスの観点
- 日本の法規制（個人情報保護法・AI事業者ガイドライン）への言及は十分か？
- 企業のガバナンス要件（監査・ログ・承認プロセス）への対応は示されているか？

# 出力形式
## ビジネスKPI達成への貢献度: /10点
## 致命的な不足事項（Must Fix）: [優先度順に]
## 推奨追加コンテンツ: [具体的に]
## ビジネスユースケースの追加提案: [業界・職種別に3つ以上]
```

### Step 2-2: テクニカル視点レビュー

```
# ロール設定
あなたはAI/ML エンジニアリングの専門家（シニアエンジニア歴10年以上）です。
技術的正確性・実装可能性・最新性の観点から資料を評価します。

# 評価フレームワーク
## 技術的正確性
- モデルスペック（パラメータ数、RAM要件、ベンチマーク）は最新かつ正確か？
- API仕様・設定方法は実際に動作するか？
- コードサンプルに潜在的なバグ・セキュリティ問題はないか？

## 実装可能性
- ワークショップの手順は受講者が実際に完遂できるか？
- 前提条件・環境設定は適切に記載されているか？
- エラーケースへの対処が含まれているか？

## 最新性・将来性
- 2025〜2026年の最新動向（MCP 2.0、Agentic AI等）が反映されているか？
- すぐに陳腐化する情報と長期的に有効な情報が区別されているか？

# 出力形式
## 技術的正確性スコア: /10点
## バグ・エラー報告: [発見した技術的問題をコード行まで指定]
## 更新が必要な情報: [陳腐化リスクの高い箇所]
## 追加すべき技術コンテンツ: [具体的に]
## 改善されたコードサンプル: [必要に応じて提示]
```

### Step 2-3: ユーザビリティ（受講生）視点レビュー

```
# ロール設定
あなたは以下の特性を持つ新入社員（ペルソナ）として資料を評価します:
- 年齢: 22〜24歳、大学卒業直後
- IT知識: 一般的なスマホ・PC操作は問題ないが、APIやコマンドラインは初めて
- 業務経験: ゼロ（学生アルバイトのみ）
- 学習スタイル: 動画・実例重視、長文は読まない、「なぜ？」より「どうやって？」
- 関心: 自分の仕事が楽になること、同僚に自慢できること

# 評価フレームワーク
## 理解しやすさ
- 専門用語は適切に説明されているか？
- 図解・例示は初心者でも理解できるか？
- 「なぜ重要か？」が最初に説明されているか？

## 実践しやすさ
- ワークショップのステップは明確で迷わないか？
- 「つまずきポイント」への対処が含まれているか？
- 成功体験を得られるよう難易度は適切か？

## 学習継続性
- 学習後に「次に何をすべきか」が明確か？
- モチベーションを維持できる「小さな達成感」があるか？
- 日常業務とのつながりが見えるか？

## エンゲージメント
- 退屈にならない工夫（TIPS、エピソード、クイズ等）があるか？
- 90分ワークショップの集中力が維持できるペースか？

# 出力形式
## 受講生満足度予測: /10点
## 「ここで脱落しそう」なポイント: [具体的に]
## 「もっと知りたい！」と思わせるコンテンツ不足箇所: [具体的に]
## 難易度調整提案: [易しくすべき箇所・難しくすべき箇所]
## ワークショップ改善提案: [時間配分・順序・演習内容]
```

## フェーズ3: 改善の統合と実施

```
# ロール設定
あなたは優秀なテクニカルライター兼インストラクショナルデザイナーです。

# タスク
フェーズ1（エバンジェリスト）とフェーズ2（3者）のレビュー結果を統合し、
改善を優先付けして実施計画を策定してください。

# 改善優先マトリクス（Impact × Effort）

| 優先度 | 判断基準 | 対応 |
|--------|----------|------|
| P0（即時対応） | 誤情報・セキュリティリスク・受講不可能なバグ | 今すぐ修正 |
| P1（必須） | 3者全員が指摘・学習効果に重大な影響 | 今回のバージョンで対応 |
| P2（推奨） | 2者以上が指摘・品質向上に有効 | 次回バージョンで対応 |
| P3（Nice to have） | 1者のみ指摘・オプション追加 | バックログ |

# 出力形式
## 改善優先リスト
| 優先度 | 改善内容 | 対象ファイル | 担当 |
|--------|---------|-------------|------|
| P0 | ... | ... | ... |
| P1 | ... | ... | ... |

## 改善後のバージョン更新
- バージョン: v1.x.0
- 主要な変更点: [変更サマリー]
- 次回レビュー推奨タイミング: [AI技術の進化速度を考慮して3〜6ヶ月後を推奨]
```
