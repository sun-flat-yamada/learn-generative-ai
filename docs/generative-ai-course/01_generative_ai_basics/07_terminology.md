# 1.7 生成AIで使われる用語と解説

> **本セクションの目標:** 生成AI関連の専門用語を正確に理解し、業務・会話の中で正確に使い分け、最新技術トレンドの文理を把握できるようになる。

---

## 本セクションの狙い

生成AIの世界では、日々新しい言葉が生まれています。これらの用語を単なる「記号」として暗記するのではなく、その技術的な背景やビジネスへの影響をセットで理解することが重要です。本セクションでは、皆さんが先輩社員や顧客との会話で自信を持ってAIを語れるよう、2026年現在の必須用語を体系的に整理します。

### 期待される学習効果

1. **共通言語の獲得:** 専門用語を正しく使うことで、開発者やAIエージェント、ビジネス部門との円滑なコミュニケーションを可能にする。
2. **モデル性能の評価眼:** トークン、コンテキストサイズ、Temperatureなどの指標を理解し、タスクに最適なAIの「設定」を自分で判断できる。
3. **最先端トレンドの把握:** 「推論モデル（Reasoning Models）」や「世界モデル（World Models）」といった2026年の主要コンセプトを理解し、AIの進化の方向性を説明できる。

## 基本用語集

### 📌 プロンプト（Prompt）

AIへの**入力指示文**のこと。人間とAIの会話における「質問」や「命令」に相当します。

```txt
【悪いプロンプト例】
「メール書いて」

【良いプロンプト例】
「以下の要件でビジネスメールを作成してください。
- 受信者: 田中部長（上司）
- 目的: 来週の会議の日程変更依頼
- トーン: 丁寧・簡潔
- 文字数: 200字以内」
```

良いプロンプトほど良い結果が得られます → [第2章: プロンプトエンジニアリング](../02_how_to_use/01_prompt_engineering.md)

---

### 📌 トークン（Token）

LLMが処理する**最小単位のテキスト**。単語・文字・サブワードなどに分割されます。

```txt
英語: "Hello, World!"  → 4 トークン
日本語: 「こんにちは」 → 5〜10 トークン（日本語は英語より多い）

目安:
- 英語: 1トークン ≈ 0.75単語
- 日本語: 1トークン ≈ 1〜2文字

OpenAI GPT-5 の料金（2026年3月目安）:
- 入力: $2.50 / 100万トークン
- 出力: $15.00 / 100万トークン
- A4 1枚 ≈ 1,000〜2,000 トークン
```

> 📚 参考:
>
> - [OpenAI Tokenizer Tool](https://platform.openai.com/tokenizer)

## 💡 TIPS: 日本語より英語の方がたくさん入って安い

> 日本語と英語で取り扱い単位が異なるため、大規模な処理を行う場合は
> 日本語より英語でプロンプトを書いた方が消費トークンが少なく済みます。
> 生成AIが受け入れることができるコンテキストサイズや利用料金でアドバンテージがあります。
> ただ、使い始めた時に気にする必要はほとんどありません。
> 不足するようになってきた時、このようなアプローチもあることを思い出してください。

### 📌 Temperature（温度パラメータ）

AIの**出力のランダム性・創造性**を調整するパラメータ（0.0〜2.0）。

```txt
Temperature = 0.0  : 決定的・一貫性あり（コード生成、翻訳に適す）
Temperature = 0.7  : バランス型（デフォルト、汎用）
Temperature = 1.0  : 自然な多様性
Temperature = 1.5  : 創造的・多彩（ブレスト、詩に適す）
Temperature = 2.0  : 非常にランダム（実用性は低い）
```

**使い分けポイント:**

- 事実の要約・翻訳 → 0.0〜0.3（低温度）
- 汎用チャット → 0.7 前後
- ブレインストーミング・創作 → 1.0〜1.5

> Temperatureの設定方法は使用するツールによって異なります。
> また、設定が公開されておらず変更できない場合もあります。

### 📌 RAG（Retrieval-Augmented Generation）

**検索拡張生成**。LLMに外部の最新情報を注入して回答精度を高める手法。2026年現在は、単なる検索だけでなくAIが状況を判断して検索結果を精査する「エージェント型RAG」が主流です。

```txt
【RAGなし】
質問: 「2025年11月のMicrosoft Igniteの発表内容は？」
回答: 「私の知識は2024年までのため、わかりません」

【RAGあり】
1. 質問を受け取る
2. 社内DB・Webを検索して関連情報を取得
3. 取得情報をプロンプトに追加（グラウンディング）
4. 最新情報に基づく回答を生成
回答: 「Work IQ、Fabric IQ、Foundry IQ が発表されました...」
```

> 📚 参考:
>
> - [AWS: What is RAG?](https://aws.amazon.com/what-is/retrieval-augmented-generation/)  
> - [IBM: Retrieval-Augmented Generation](https://www.ibm.com/think/topics/retrieval-augmented-generation)

### 📌 MCP（Model Context Protocol）

Anthropic が提唱し、業界標準になりつつある**AIとツール・データソースを接続するオープンプロトコル**。

```txt
【MCPなし】
AI → 社内ツールごとに個別API実装 → 拡張性が低い

【MCPあり】
AI（Claude, GPT, Copilot 等）
    ↕ (MCPプロトコル)
[MCPサーバー群]
├── ファイルシステム
├── GitHub
├── Slack
├── 社内データベース
└── 任意のカスタムツール
→ 標準化されたインターフェースで接続
```

→ [第2章 2.3: ナレッジの蓄積と活用（MCPサーバー）](../02_how_to_use/03_knowledge_mcp.md)

### 📌 推論モデル（Reasoning Models）

2025年の「o1」登場以降、一般化したモデルの分類。従来の「次の単語を予測する（高速）」だけでなく、**「内部で筋道を立てて考える（思考ステップを持つ）」**ことに特化したAI。

- **System 2思考:** 「即答」ではなく、内部で何度も自問自答を繰り返してから回答を出力します。
- **適したタスク:** 数学、複雑なコーディング、論理パズル、科学的発見。
- **主なモデル:** GPT-5 (Thinking mode), Claude 4.x (Logic optimized), DeepSeek R1。

---

### 📌 世界モデル（World Models）

テキストだけでなく、**「物理的な世界の仕組み」を理解・シミュレーションしようとするAI**。

- **何をするのか:** 「リンゴを離せば下に落ちる」「液体は形を変える」といった物理法則を動画や3Dで予測します。
- **インパクト:** 現実そっくりの動画生成（Sora, Genie 3）や、ロボットが未経験の行動を自律的に学習する基盤となります。
- **代表的な開発元:** Google DeepMind, OpenAI, World Labs, Meta。

### 📌 Agentic AI（エージェントAI / AIエージェント）

指示を待つだけの「チャットボット」から、**「目標を与えれば、自律的にツールを使い、手順を考えて実行する」**存在への進化。

- **自律動作:** 自分でブラウザを開き、航空券を予約し、日程をカレンダーに入れるといった複数ステップの実行。
- **共創:** 単なる道具ではなく、チームメイトの一人として振る舞います。
- **関連技術:** [MCP（Model Context Protocol）](./03_knowledge_mcp.md) 等の接続規格が、エージェントがツールを使うための「手足」となります。

### 📌 その他の重要用語

| 用語                 | 説明                                                                             |
| -------------------- | -------------------------------------------------------------------------------- |
| **LLM**              | Large Language Model（大規模言語モデル）                                         |
| **SLM**              | Small Language Model（小規模モデル）。モバイル端末等で高速・安全に動く小型AI     |
| **Hallucination**    | AIが事実と異なる情報を生成する現象。2026年も重要課題                             |
| **RLHF**             | Reinforcement Learning from Human Feedback（人間のフィードバックによる強化学習） |
| **System Prompt**    | ユーザーに見えない形でAIの動作を設定する命令文                                   |
| **Zero-shot**        | 事例なしで指示だけでタスクを実行させる手法                                       |
| **Few-shot**         | 少数の例示（Example）を与えてタスクを実行させる手法                              |
| **Chain-of-Thought** | AIに「段階的に考えさせて」から答えを出させる手法（思考の連鎖）                   |
| **Synthetic Data**   | AI自身が生成したデータ（合成データ）。学習データの不足を補うために使われる       |
| **Multimodal**       | テキスト、画像、音声、動画、3Dをシームレスに扱える能力                           |
| **Context Window**   | AIが一度に覚えられる記憶の大きさ。Gemini 3 等では200万トークンを超える           |
| **Fine-tuning**      | 既存のAIを、特定の業務や専門知識に合わせて「追加教育」する手法                   |
| **Grounding**        | AIの回答を、外部の信頼できる事実やデータに「着地」させて精度を高めること         |
| **Quantization**     | モデルのパラメータ精度を下げてメモリを節約する技術                               |
| **Embedding**        | 言葉を「数字の列（ベクトル）」に変換し、AIが意味の近さを計算できるようにする技術 |
| **Vector DB**        | Vector Database（Embeddingを保存・検索するためのデータベース）                   |
| **Green AI**         | AIの計算コストや電力消費を抑え、環境に配慮したサステナブルなAI開発・利用         |
| **Agentic AI**       | 自律的にタスクを計画・実行するAI                                                 |

## 💡 TIPS: 「知っておくと会話が弾む」業界トリビア

> - **「Attention is All You Need」**: 2017年に発表された、現代AI（Transformer）のすべての始まりとなった論文。
> - **「LLM はオートコリクトの神版」**: 「次に続く最も確率が高い文字」を予測しているだけというLLMの本質を突いた表現。
> - **「Vibe Coding」**: Andrej Karpathyが提唱。「細かいコードは書かず、AIに『感じ（Vibe）』を伝えて作らせる」新時代の開発スタイル。
> - **「AGI（汎用人工知能）」**: 人間と同等、あるいはそれ以上の知能を持つ自律的AI。2026年現在はその入り口に立っていると言われています。
> - **「Scale is all you need」**: パラメータ数と学習データを増やせばAIは賢くなる、という（かつての）経験則  
> - **「AI Winter」**: AIへの期待が冷め、投資や研究が停滞した時代（1970年代、1980〜90年代）

1. **RAGのたとえ話:** 「試験中に教科書を持ち込める状態」や「図書館で調べながら答える秘書」など、相手がイメージしやすい例えを作ってみる。
2. **エージェントAIのたとえ話:** 「ただ質問に答えるだけの辞書」と「自分で考えて買い物に行ってくれるお手伝いさん」の違いとして伝えてみる。

> 自分ごととして捉えることで、技術の理解が圧倒的に深まります。
