# 2.1b 補講：プロンプト逆引き辞典（Prompt Cookbook）

> 「どう書けばいいかわからない」を解消するコピペで使える逆引き集です。

---

## 使い方

各プロンプトの `【　】` 内を自分の状況に合わせて書き換えて使用してください。  
ChatGPT・Claude・Gemini・GitHub Copilot で動作確認済みです。

## 🧠 推論モデル（Reasoning / Deep Thinking）系

複雑な論理パズル、システム設計、深い分析が必要な際に使用します。

### ① 複雑な問題の多角的分析（o1/o3, Claude 3.7, Gemini 2.5 理論）

```txt
<task>
以下の課題について、複数の視点（技術、ビジネス、ユーザー、法的制約）から深く推論し、最適な解決策を提案してください。
</task>

<problem>
【課題の具体的な内容をここに記載】
</problem>

<constraints>
- 結論を出す前に、それぞれの視点でのメリット・デメリットを自己批判的に検討すること。
- 実行可能なステップに分解して出力すること。
</constraints>
```

## 📝 文書・資料作成系

### ② エグゼクティブ・サマリー（要約）

```txt
<task>
添付（または以下）の資料を、多忙な役員が1分で理解できるように要約してください。
</task>

<source_text>
【ここに長い文章を貼り付け、またはファイルを添付】
</source_text>

<format>
1. 結論（何が最重要か）
2. 3つの重要ポイント（箇条書き）
3. ネクストアクション
</format>
```

### ③ 会議議事録の構造化

```txt
以下の会議メモから正式な議事録を作成してください。

<memo>
【会議メモをここに貼り付け】
</memo>

<format>
## 会議概要
- 日時：
- 参加者：

## 議論の要点
（議題ごとに整理）

## 決定事項
（箇条書き）

## アクションアイテム
| 担当 | 内容 | 期限 |
| ---- | ---- | ---- |
</format>
```

## 🖼️ マルチモーダル系（画像・動画・音声）

2026年現在のAIは「目」と「耳」を持っています。

### ④ スクリーンショットからの実装生成

（UIデザイン案やホワイトボードの写真をアップロードして使用）

```txt
<task>
アップロードした画像を分析し、これと全く同じ見た目のUIを【React + Tailwind CSS / Flutter / etc.】で実装してください。
</task>

<requirements>
- レスポンシブ対応にすること
- モダンで清潔感のあるデザインを維持すること
- 各パーツのアクセシビリティも考慮すること
</requirements>
```

### ⑤ 手書きメモのデジタル化

```txt
<task>
この画像に写っている手書きのメモをテキストデータに書き起こしてください。
もし図解がある場合は、その内容を説明するか、Mermaid記法で図式化してください。
</task>
```

## 💻 コーディング & エージェント系

### ⑥ コード生成 & セルフレビュー（エージェント的アプローチ）

```txt
<task>
以下の機能を実装してください。
実装後、自らコードをレビューし、バグやパフォーマンスの懸念点がないか確認した上で最終案を提示してください。
</task>

<requirements>
- 言語: 【TypeScript】
- 機能: 【具体的な機能説明】
</requirements>

<style_guide>
- [Role] および [Mechanism] コメントを付与すること
- ユニットテストを含めること
</style_guide>
```

### ⑦ データ抽出（構造化出力）

```txt
<task>
以下の非構造化テキストから、必要な情報を抜き出し、有効なJSON形式で出力してください。
</task>

<source>
【レシート、契約書、メールなどのテキスト】
</source>

<json_schema>
{
  "date": "YYYY/MM/DD",
  "total_amount": number,
  "items": [{"name": string, "price": number}]
}
</json_schema>
```

## 💡 TIPS — 2026年のプロンプトのあり方

> **「文章」から「プロセス」へ**
>
> かつて、米国では「プロンプトエンジニア」という職種が年収 **17.5万ドル（約2,600万円）** で募集されたことがありました（Anthropic 社, 2023年）。
>
> 2023年頃までは「魔法のキーワード」を探すことが重視されていましたが、2026年現在は**「AIをどう動かすか」というワークフロー設計（Context Engineering）**が重要です。
> AI自体の理解力が向上したため、単なる「文章のコツ」よりも、**「どの業務にどうAIを組み込むか」という設計能力**が重視されています。
> 「魔法の言葉」を探すより、**明確な指示を出す能力**を磨きましょう。
>
> ソフトウエア開発の歴史を振り返ってみても、実現したいことを分析、整理して、コンピューターが理解する形(言語)に合わせて再構築する能力が、常に重要視されてきました。プロンプトエンジニアリングも、その延長線上にあると言えるでしょう。

1. **XMLタグの活用:** `<context>`, `<task>`, `<constraints>` のように構造化することで、AIは迷わず指示に従えます。
2. **AIに「考えさせる」:** 推論モデルに対しては、詳細なステップを指定するよりも「まず自分で計画を立ててから実行して」と、高い抽象度で任せるほうが良い結果が得られます。
3. **AIを「エージェント」として扱う:** 単なる検索ツールではなく、計画・実行・評価を行うパートナーとして指示を出すことが、今の時代の標準（Agentic AI）です。
