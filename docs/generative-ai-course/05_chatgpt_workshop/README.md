# 第5章: ChatGPT 活用ワークショップ (2026年版)

> **形式:** ハンズオン（グループワーク推奨）  
> **所要時間:** 約90分  
> **準備:** ChatGPT アカウント（Plus/Pro 推奨、最新の o1 または GPT-5.x モデルが利用可能であること）

---

## ワークショップ概要

2026年現在、ChatGPT は単なる「チャットボット」から、自律的に思考し、調査し、人間と共創する **「AI エージェント / 思考パートナー」**へと進化しました。このワークショップでは、最新の3つの主要機能（Reasoning, Search, Canvas）を使いこなすための3つのお題に挑戦します。

```text
お題1: 【共創】Canvas を使った「ビジネスドキュメントの共同執筆」（30分）
お題2: 【調査】Search 連携による「市場動向のリアルタイム分析」（30分）
お題3: 【思考】o1 推論モデルによる「複雑な業務プロセスの最適化」（30分）
```

---

## お題1: Canvas を使った「ビジネスドキュメントの共同執筆」

### 【2026年の鳥肌ポイント】
これまでの AI は「出力して終わり」でした。現在は **Canvas** インターフェースにより、AI と人間が一つのドキュメントを並んで編集できます。AI がドラフトを書くだけでなく、あなたが書いたものに対して「特定の行への修正案」を出したり、「絵文字を追加」「トーンを変える」といった指示を特定の範囲だけに適用できるようになりました。

### シナリオ（お題1）

あなたは新サービスのプロジェクトリーダーです。以下の状況で、経営陣向けの「プロジェクト進捗報告（悪いニュースを含む）」のドラフトを作成し、ブラッシュアップしてください。

```text
状況:
- プロジェクト: AI搭載型在庫管理システム「SmartStock」
- 課題: 開発フェーズで予期せぬ不具合が見つかり、リリースが2週間遅延する
- 要点: ただし、この遅延を利用して最新の o1 推論エンジンを統合し、精度を15%向上させる計画
- アクション: 来週の役員会議で説明するための骨子が必要
```

### ステップ1: Canvas モードでのドラフト生成

以下のプロンプトを入力し、出力が **Canvas**（チャット欄の横にエディタが開く形式）で表示されることを確認してください。

```markdown
# Role
あなたは経験豊富なプロジェクトマネージャーです。

# Task
役員向けのプロジェクト進捗報告の初回ドラフトを、Canvas 形式で作成してください。

# Requirement
- リリースの2週間延期というネガティブな情報を、戦略的なメリット（推論エンジン統合による精度向上）としてポジティブに転換する構成にすること。
- 丁寧だが過度に卑屈にならない、プロフェッショナルなトーン。
```

### ステップ2: Canvas 上での編集・ブラッシュアップ

Canvas が開いたら、以下の操作を試してください。

1. **「編集を依頼」機能:** Canvas 内の特定の段落を選択し、「ここをもっと具体的な技術的メリットに書き換えて」と指示する。
2. **ショートカットの活用:** 「長さの調整」や「磨きをかける」ボタン（Canvas 機能）を使って、全体を洗練させる。

## お題2: Search 連携による「市場動向のリアルタイム分析」

### 【2026年の鳥肌ポイント】
ChatGPT は最新のインターネット情報を自ら検索できるようになりました（**Search** 機能）。統計データ、ニュース、競合の最新プレスリリースをリアルタイムで収集し、それらを組み合わせて「今、この瞬間の分析」を行うことができます。

### シナリオ（お題2）

あなたはマーケティングリサーチ担当です。新年度の戦略立案のため、「日本国内の生成AIトレンド（2026年3月現在）」を調査し、競合他社の動きと比較してください。

### ステップ（お題2）

1. **Search モードの起動:** チャット欄の「検索（地球儀アイコン）」をクリックするか、明示的に「最新情報を検索して」と伝えます。
2. **複合プロンプトの実行:**

```text
「2026年3月現在の日本における、企業の生成AI導入に関する最新統計を検索してください。
また、主要なIT企業の最近の動向（特にエージェント活用事例）を3社ピックアップし、
自社の強み（誠実な顧客対応、地域密着）を活かせる参入領域を提案してください。」
```

**観察:** Google 検索の結果をただ要約するのではなく、あなたの「自社の強み」という文脈と組み合わされた提案が出てきましたか？

## お題3: o1 推論モデルによる「複雑な業務プロセスの最適化」

### 【2026年の鳥肌ポイント】
**o1 (Reasoning Model)** は、生成の前に「思考（Chain-of-Thought）」を数秒〜数十秒間行います。これにより、従来のモデルでは苦手だった「論理的矛盾の解消」「多段階の計算」「複雑な条件分岐」が含まれる問題を自律的に考え抜くことができます。

### シナリオ（お題3）：物流・リソース最適化問題
以下の複雑な状況下で、最も効率的な翌日の配送ルートと人員配置を計画してください。

```text
制約条件:
- 配送車: 3台（A: 10t, B: 4t, C: 2t）
- 配送先: 15拠点（各々で荷物の量と時間指定が異なる）
- ドライバーの休憩規制: 連続4時間運転で30分休憩必須
- 道路状況: 特定の国道が工事中で、大型車（A）は迂回が必要
- コスト優先度: 燃料費（距離）の最小化 > 時間の遵守
```

### ステップ1: o1 モデルの選択
ChatGPT のモデル選択から **o1** (または最新の推論モデル) を選択します。

### ステップ2: 思考プロセスを観察する
プロンプトに上記条件を入力し、実行します。

**注目ポイント:**

- 「Thinking...（思考中）」の表示をクリックして、AI がどのように条件を整理し、矛盾を排除しようとしているかの過程を見てください。
- 従来の GPT-4 以前のモデルであれば「それっぽいが計算が合わない」答えが出やすかったですが、o1 は論理的に破綻のない計画を提示できるはずです。

## ワークショップ まとめ

```text
2026年の ChatGPT 活用 4大ルール:
1. 「書かせる」から「一緒に育てる」へ (Canvas)
2. 「知識の検索」から「文脈を伴う調査」へ (Search)
3. 「即答」を求めず「深く考えさせる」 (o1 Reasoning)
4. AI の出力を鵜呑みにせず、最後の「意志決定」は人間が行う
```

**次のアクション:**
→ 自身の業務の中で、「一回のやり取りでは終わらない、複雑なドキュメント作成」はないか？ (Canvas の出番)
→ 「AI に聞いても『2023年までの知識です』と言われる」と諦めていた調査はないか？ (Search の出番)
→ 「論理的に難しすぎて AI には無理だ」と思っていた業務改善案はないか？ (o1 の出番)
